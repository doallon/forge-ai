# AI Framework Templates

## Purpose

This directory contains reusable output contracts for Forge AI and AI Framework adopters.

Templates define structure, required sections, evidence expectations, review readiness, and certification support.

Templates do not define architecture by themselves. They consume AI Framework authority.

## Category Responsibilities

| Category | Responsibility |
| --- | --- |
| Core | Template authoring rules, metadata, naming, cross-reference standards. |
| Planning | Vision, roadmap, phase, stage, and historical capability planning. |
| Architecture | Architecture, RFC, ADR, specification, and governance outputs. |
| Workflow | Workflow, command, checklist, review, approval, decisions, handoffs. |
| Validation | Validation, audit, evidence, quality gates, risk, compliance, certification. |
| Knowledge | Persistent knowledge architecture, registry, and source records. |
| Memory | Derived reusable learning and memory governance. |
| Context | Task context assembly, packets, and context engine contracts. |
| Runtime | Runtime context and decision context. |
| Agents | Single-agent role, task, capability, execution, and handoff contracts. |
| MultiAgent | Multi-agent coordination, synchronization, and conflict reporting. |
| Swarm | Swarm planning, decomposition, merge, execution, and review. |
| Platform | Adapter specification, boundaries, capability map, and validation. |
| Integration | External integrations, connectors, registries, and tool contracts. |
| Migration | Migration plans, reports, compatibility matrix, deprecation notices. |
| Project | Generic and framework-specific project adoption templates. |

## Core Rule

A template defines output structure. It does not override:

```text
AGENTS.md
docs/AI/AIFramework.md
docs/AI/Specification/Constitution.md
docs/AI/AIOrchestrator.md
docs/ProjectStatus.md
```
