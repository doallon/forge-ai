# AI Templates

---

## Purpose

This directory contains the canonical reusable document templates used by Forge AI.

Templates standardize document structure, metadata, evidence presentation, review readiness, and certification preparation across the repository.

Templates are reusable output contracts.

They are **not** architectural authorities.

They consume the approved governance and architecture model.

---

## Authority Consumption

Templates consume the current Forge AI authority hierarchy.

They shall never redefine or replace it.

```text
Human Governance
        ↓
AGENTS.md
        ↓
docs/AI/GOVERNANCE.md
        ↓
docs/FrameworkGovernance.md
        ↓
docs/DevelopmentPhases/ProjectStatus.md
        ↓
docs/DevelopmentPhases/ForgeAI-DevelopmentPhases.md
        ↓
Constitution
        ↓
Meta Models
        ↓
Standards
        ↓
Runtime
        ↓
Engine Platform
        ↓
Operational Core
        ↓
Templates
```

Templates consume authority.

Templates never become authority.

## Category Responsibilities

| Category | Responsibility |
|-----------|----------------|
| Core | Template authoring rules, metadata, naming conventions, and shared template standards. |
| Planning | Vision, roadmap, phase, stage, milestone, and planning artifacts. |
| Architecture | RFC, ADR, architecture specifications, governance artifacts, and design documents. |
| Workflow | Commands, workflows, reviews, approvals, handoffs, and operational procedures. |
| Validation | Validation, verification, audit evidence, compliance, certification support, and quality gates. |
| Knowledge | Knowledge artifacts, registries, discovery outputs, and knowledge lifecycle templates. |
| Memory | Persistent memory structures and reusable learning templates. |
| Context | Context packets, execution context, and Context Engine support templates. |
| Runtime | Runtime execution, lifecycle, and execution-state templates. |
| Agents | Single-agent execution, capabilities, responsibilities, and handoff templates. |
| MultiAgent | Multi-agent collaboration, synchronization, decomposition, and merge templates. |
| Swarm | Swarm planning, execution, review, aggregation, and coordination templates. |
| Platform | Platform adapters, integrations, capabilities, and boundary specifications. |
| Integration | External connectors, registries, adapters, interoperability, and API templates. |
| Migration | Migration plans, compatibility matrices, deprecation notices, and transition reports. |
| Project | Project-level planning and adoption templates. |

## Operational Rules

Templates define:

- document structure
- mandatory sections
- metadata layout
- evidence expectations
- completion expectations
- review readiness

Templates do **not** define:

- governance
- roadmap
- architecture
- runtime
- engine behavior
- operational policy
- implementation
- certification
- canonical promotion

## Operational Boundary

Templates are reusable artifacts.

Templates never become:

- Source of Truth
- Runtime
- Engine
- Governance
- ProjectStatus
- Roadmap
- Operational Core

They exist solely to standardize repository outputs.
