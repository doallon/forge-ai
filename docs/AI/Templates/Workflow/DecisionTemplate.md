# Purpose

This template defines the required structure for operational decision records.

Use this template for workflow, process, execution, tooling, or governance-routing decisions. Use `ADRTemplate.md` for architecture decisions.

# Required Structure

## 1. Decision Identification

Decision ID, title, date, owner, authority, and status.

## 2. Decision Context

Explain the problem, trigger, or decision point.

## 3. Options Considered

List considered options and tradeoffs.

## 4. Decision

State the selected decision clearly.

## 5. Rationale

Explain why the decision was selected.

## 6. Impact

Document effects on workflow, command routing, validation, review, certification, project state, or tooling.

## 7. Constraints

List boundaries and non-negotiable rules preserved by the decision.

## 8. Follow-up Actions

List required implementation, documentation, validation, or review work.

## 9. Review / Expiry

State whether the decision is permanent, temporary, or scheduled for review.

# Completion Checklist

- [ ] Context defined
- [ ] Options considered
- [ ] Decision stated
- [ ] Rationale documented
- [ ] Impact documented
- [ ] Constraints preserved
- [ ] Follow-up actions listed
- [ ] Review or expiry status defined
