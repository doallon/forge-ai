# Purpose

This template defines the required structure for handoff records between agents, humans, workflows, stages, or tools.

A handoff preserves continuity. It does not certify work by itself and does not update project state unless the approved state workflow permits it.

# Required Structure

## 1. Handoff Identification

Handoff ID, source actor, target actor, date, related task/capability/stage, and status.

## 2. Context Summary

Summarize the completed work and current situation.

## 3. Authority and Scope

List governing documents and active scope boundaries.

## 4. Completed Work

List completed artifacts, files, documents, or decisions.

## 5. Validation Evidence

List validation run, results, blocked gates, or pending validation.

## 6. Review / Certification Status

State whether review or certification has occurred, is pending, or is blocked.

## 7. Open Items

List unresolved items, blockers, risks, and deferred work.

## 8. Next Recommended Action

State the next action and the workflow/command that should handle it.

## 9. Files / Artifacts

List relevant paths and artifacts.

## 10. Warnings

List constraints the next actor must preserve.

# Completion Checklist

- [ ] Context summarized
- [ ] Authority listed
- [ ] Scope boundaries clear
- [ ] Completed work listed
- [ ] Validation evidence included
- [ ] Review/certification status stated
- [ ] Open items documented
- [ ] Next action recommended
- [ ] Relevant artifacts listed
- [ ] Warnings included
