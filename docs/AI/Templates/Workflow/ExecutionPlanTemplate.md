# Purpose

This template defines the required structure for execution plans.

An execution plan turns approved scope into ordered work. It does not create new architecture and does not replace phase, stage, or capability planning.

# Required Structure

## 1. Execution Plan Identification

Plan ID, title, owner, date, related phase/stage/capability, authority, and status.

## 2. Objective

Define the execution objective.

## 3. Scope

Define included work, excluded work, and explicit non-goals.

## 4. Preconditions

List required documents, approvals, inputs, dependencies, and environment conditions.

## 5. Work Breakdown

List ordered execution steps.

| Step | Action | Owner | Inputs | Outputs | Validation |
| --- | --- | --- | --- | --- | --- |

## 6. Dependency Order

Explain sequencing and blocking dependencies.

## 7. Validation Plan

Define evidence, commands, quality gates, and expected outcomes.

## 8. Review Plan

Define review owner, checklist, and readiness criteria.

## 9. Rollback / Recovery

Define rollback, recovery, or blocker handling.

## 10. Completion Criteria

Define what makes the plan complete.

## 11. Handoff

Define next actor, next workflow, or next state.

# Completion Checklist
