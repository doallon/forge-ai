## 5. Preparation Package

### 5.1 External Target Authority Requirements

Axis Suite external Target execution requires explicit Axis Suite-owned authority before work may proceed. At minimum, the external Target must provide or identify:

| Required Declaration | Preparation Requirement |
|:---|:---|
| Target repository root | Explicit Axis Suite repository root for all path resolution. |
| Root Target contract | Axis Suite root `AGENTS.md` or equivalent Target contract governing AI-DOS operation. |
| Operational state path | Axis Suite-owned ProjectStatus or equivalent live state path, relative to Axis Suite root. |
| Planning / phase authority | Axis Suite-owned DevelopmentPhases, roadmap, or equivalent capability direction path, relative to Axis Suite root. |
| Architecture authority | Axis Suite architecture source(s), if relevant to the bounded pilot. |
| Source scope | Explicit source roots or files that may be inspected or modified. |
| Protected areas | Axis Suite protected paths, read-only areas, approval gates, and forbidden changes. |
| Validation commands | Axis Suite validation commands, expected working directory, dependencies, environment limits, and pass/fail interpretation. |
| Evidence expectations | Required execution evidence, review evidence, blocker evidence, and completion report requirements. |
| Authority order | Conflict-resolution order among Axis Suite instructions, AI-DOS provider instructions, and Human Governance instruction. |
| AI-DOS consumption rule | Explicit statement that AI-DOS provides reusable operating behavior while Axis Suite owns project truth. |

### 5.2 Bounded Pilot Scope

The first Axis Suite external Target pilot should be bounded to one Human-Governed work unit only. The pilot must not begin until Human Governance supplies explicit external Target authority. The pilot scope should include:

1. Read Axis Suite Target contract and declared authority files.
2. Resolve exactly one Axis Suite active task or authorized next action.
3. Confirm Axis Suite protected areas and validation requirements.
4. Execute only the selected Axis Suite work unit if separately authorized.
5. Produce evidence proving Target isolation, validation results, protected-area preservation, and no Forge AI path fallback.
